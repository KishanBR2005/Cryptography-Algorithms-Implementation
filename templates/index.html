AES (Advanced Encryption Standard)
Symmetric Encryption

AES is a symmetric encryption algorithm, meaning the same key is used for both encryption and decryption.

  • Block Size: 128 bits
  • Key Size: 128, 192, or 256 bits (we use 256 for maximum security)
  • Mode: GCM (Galois/Counter Mode) for authenticated encryption
  • Speed: Fast, suitable for large data
  • Use Case: Encrypting sensitive data like documents, messages, files

How it works: You encrypt data with a secret key, and only someone with that same key can decrypt it.

RSA (Rivest-Shamir-Adleman)
Asymmetric Encryption

RSA uses a public-private key pair. The public key encrypts data, and only the private key can decrypt it.

  • Key Pair: Public (share with everyone) and Private (keep secret)
  • Key Size: 2048 or 4096 bits (we use 2048 for balance)
  • Mode: OAEP (Optimal Asymmetric Encryption Padding)
  • Speed: Slower than AES, but enables secure key exchange
  • Limitation: Can only encrypt ~245 bytes with 2048-bit key
  • Use Case: Secure communication, digital signatures, key exchange

How it works: Anyone can send you encrypted messages using your public key, but only you can decrypt them with your private key.

SHA-256 (Secure Hash Algorithm)
Cryptographic Hash Function

SHA-256 converts any input into a fixed 256-bit (64 hex character) output. It's one-way and collision-resistant.

  • Output Size: 256 bits (64 hexadecimal characters)
  • Reversible: No - you cannot get the original input from the hash
  • Deterministic: Same input always produces same hash
  • Collision Resistance: Extremely hard to find two inputs with same hash
  • Use Case: Password storage, data integrity, digital signatures

For Passwords: We use salting (random data) to prevent rainbow table attacks. Each password gets a unique salt and hash.

For Data Integrity: Hash files or messages to verify they haven't been tampered with.

Getting Started
Step 1: AES Encryption
  1. Click "Generate New AES Key" to create a secret key
  2. Enter text in the plaintext field
  3. Click "Encrypt" to get ciphertext, nonce, and tag
  4. To decrypt, paste the key, ciphertext, nonce, and tag into the decryption form
Step 2: RSA Encryption
  1. Click "Generate New Keypair" to create public and private keys
  2. Share the public key with others (it's safe to share)
  3. Keep the private key secret!
  4. To encrypt: use the public key to encrypt a message
  5. To decrypt: use the private key to decrypt a message
Step 3: Password Hashing
  1. Enter a password and click "Hash Password"
  2. Store the "Full Hash" in your database (includes salt)
  3. To verify: enter the password and stored hash, click "Verify Password"
  4. The system hashes the input with the same salt and compares the results
Security Notes
  • RSA Private Keys: Never share or expose your private keys. They're like passwords for your encrypted data.
  • AES Keys: Treat AES keys as secrets. They should be stored securely, not in plain text.
  • Password Hashing: Always use salt when hashing passwords. Never store plain-text passwords.
  • Key Size: 2048-bit RSA and 256-bit AES are considered secure for most use cases.
  • HTTPS: In production, always use HTTPS to encrypt data in transit.
  • This Demo: This is a learning tool. For production use, follow proper security practices.
